# Aquora — Pool, Spa & Fountain Equipment (UAE)

> **▶ Current state & operations:** see **[HANDOFF.md](HANDOFF.md)** — the platform has since grown to **5,935 real products with photos and full spec tables**, **Google Retail semantic search**, faceted search + sort, a **multimodal (text + photo) Gemini assistant**, and complete SEO/structured data. This README documents the original v1 architecture.

A complete headless e-commerce business built with **MedusaJS v2** (backend) + **Next.js 15** (storefront), with an optional **Google Gemini** on-site advisor. Aquora is an original premium Dubai/GCC brand selling pool, spa, pond and fountain equipment. The information architecture and feature set are modelled on a reference site (waterstore.ae) but **all branding, copy, catalogue and imagery are original** — nothing is copied.

> Built autonomously: brand system, a 96-product AED catalogue (12 categories), and all marketing copy were generated by parallel AI agents; the Medusa backend and Next storefront were scaffolded, seeded and themed end-to-end.

---

## Architecture

```
aquora/
├── docker-compose.yml          # Postgres 16 + Redis 7
├── data/                       # Generated source-of-truth (brand, catalog, content, research)
└── aquora-store/               # Medusa v2 monorepo (turbo + npm workspaces)
    └── apps/
        ├── backend/            # Medusa server + Admin (http://localhost:9000, admin at /app)
        │   └── src/
        │       ├── data/aquora-catalog.json     # 12 categories × 8 products
        │       ├── scripts/aquora-seed.ts        # AED/UAE seed (categories, products, region, shipping)
        │       └── api/store/assistant/route.ts  # Gemini-backed "Ask Aqua" endpoint
        └── storefront/         # Next.js 15 storefront (http://localhost:8000)
            └── src/lib/aquora/  # brand.ts, categories.ts, content/ (pages + blog)
```

- **Database:** Postgres (Docker). **Cache/events:** Redis (Docker).
- **Currency / region:** AED, region "United Arab Emirates" (ships across the GCC: ae, sa, qa, kw, bh, om).
- **Payments:** dev/manual provider (`pp_system_default`) works out of the box — no keys needed to complete a test checkout. Stripe is wired behind an env var for production.
- **AI:** `/store/assistant` answers product questions. Without `GEMINI_API_KEY` it returns keyword-matched product suggestions; with a key it uses Gemini for natural answers grounded in the catalogue.

## Prerequisites

- Docker Desktop (running)

## Run it

```bash
# 1. Start the database layer
cd aquora
docker compose up -d

# 2. Backend (Medusa) — http://localhost:9000  (admin at /app)
cd aquora-store/apps/backend
npx medusa develop

# 3. Storefront (Next.js) — http://localhost:8000   (in a second terminal)
cd aquora-store/apps/storefront
npm run dev
```

**Admin login (dev):** `admin@aquora.ae` / `Aquora!2026` → http://localhost:9000/app

### Reseed the catalogue (clean)

```bash
cd aquora-store/apps/backend
docker exec aquora-postgres psql -U postgres -c "DROP DATABASE IF EXISTS aquora WITH (FORCE);" -c "CREATE DATABASE aquora;"
npx medusa db:migrate
npx medusa exec ./src/scripts/aquora-seed.ts     # prints the storefront PUBLISHABLE_KEY
npx medusa user -e admin@aquora.ae -p 'Aquora!2026'
```

The seed prints a `PUBLISHABLE_KEY=pk_…`. Put it in `apps/storefront/.env.local` as `NEXT_PUBLIC_MEDUSA_PUBLISHABLE_KEY`.

## Environment / API keys

The build runs fully in dev **without** any external keys. To go live, set:

| Key | Where | Enables |
|-----|-------|---------|
| `GEMINI_API_KEY` | `apps/backend/.env` | Natural-language answers in the "Ask Aqua" assistant (Google AI Studio key). Optional: `GEMINI_MODEL` (default `gemini-flash-latest`). |
| `STRIPE_API_KEY` + `NEXT_PUBLIC_STRIPE_KEY` | backend `.env` / storefront `.env.local` | Real card payments (see below). |
| `FIRECRAWL_API_KEY` | `aquora/.env.local` | Reference-site crawling for research (build-time only). |

### Enable Gemini
Add `GEMINI_API_KEY=...` to `apps/backend/.env` and restart the backend. The assistant route auto-detects it and switches from keyword fallback to Gemini.

### Enable Stripe (production payments)
1. `cd apps/backend && npm i @medusajs/payment-stripe`
2. Add a Payment module provider in `medusa-config.ts` gated on `STRIPE_API_KEY`.
3. Add the Stripe provider to the UAE region's `payment_providers`.
4. Set `NEXT_PUBLIC_STRIPE_KEY` in the storefront.

## Catalogue

12 categories × 8 products = **96 SKUs**, all original "Aquora house line" products with AED pricing:
`pool-filtration, pool-pumps, pool-heating, pool-cleaning, water-treatment, pool-lighting, spa-wellness, fountains, ponds, pipes-fittings, covers-reels, accessories`.

Edit `apps/backend/src/data/aquora-catalog.json` and reseed to change the catalogue.

> **Product images:** not included yet (the storefront shows placeholders). Add real photography or generated imagery and attach via the Admin or by adding `images: [{ url }]` in the seed.
